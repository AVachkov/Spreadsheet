# Електронна таблица (Spreadsheet)

Конзолно приложение за работа с електронни таблици (*Console Line Interface*), разработено на *C++* като курсов проект по Обектно-ориентирано програмиране във ФМИ, СУ.

## Поддържана функционалност

* **Файлови операции:** Четене и запис на таблици от текстови файлове.
* **Типове клетки:** Поддръжка на цели числа, дробни числа, текст и математически формули.
* **Изчисляване на формули:** Автоматично оценяване на базови аритметични операции (събиране, изваждане, умножение, деление, степенуване).
* **Връзки между клетки:** Рефериране (препратки) към други клетки в матрицата (напр. `=R1C1 + R2C2`).
* **Валидация:** Автоматично засичане на циклични зависимости (рекурсивно извикване на клетки) и невалидни данни.

## Компилация и стартиране

### Вариант 1: Чрез терминал (Препоръчително)

Отворете терминала в главната директория на проекта и изпълнете съответната команда за Вашата операционна система:

**За *Mac* / *Linux*:**

```bash
clang++ -std=c++17 -Iinclude src/*.cpp -o build/main
./build/main

```

**За *Windows*:**

```cmd
g++ -std=c++17 -Iinclude src\*.cpp -o build\main.exe
build\main.exe

```

### Вариант 2: Чрез *Visual Studio Code*

Проектът съдържа конфигуриран файл *.vscode/tasks.json*. За автоматично компилиране натиснете:

* *Cmd + Shift + B* (за *Mac*)
* *Ctrl + Shift + B* (за *Windows* / *Linux*)

## Примерни тестове

В директорията `tests/` са подготвени примерни текстови файлове с валидни и некоректни данни (`test_basic.txt`, `test_formulas.txt`,...), с които можете да тествате поведението на командите *open*, *print*, *edit* и *save*.

## Декларация за използван външен код

Заявявам, че проектът е разработен самостоятелно, с изключение на следния специфичен фрагмент:

* **Файл:** *src/cell.cpp*
* **Метод:** *TextCell::getContentLength()*
* **Описание:** Логиката, използваща побитови маски за правилно изчисляване на дължината на символен низ в *UTF-8* формат (за коректно подравняване на текст на кирилица при отпечатването на таблицата), е реализирана с помощта на генеративен модел. Подходът е използван, тъй като стандартната функция *text.length()* връща броя на байтовете, а не на реалните символи, което нарушава визуализацията на таблицата при използване на кирилица.
